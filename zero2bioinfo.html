Logicals

A logical can have the values TRUE, FALSE, and NA (Not Available). You might see TRUE and FALSE abbreviated as T and F, respectively, but I dont recommend using the abbreviation. You can create logicals with: <, <=, >, >=, == for equality, != for inequality. Logical operators are: ! for not; & for and; | for or.

4==4 #is 4 equal to 4?
## [1] TRUE
4!=5 #is 4 not equal to 5?
## [1] TRUE
x <- 4 > 5 #assign the value of 4>5 (i.e. FALSE) to x
!x #not x
## [1] TRUE
x | (5>4)
## [1] TRUE
x & TRUE
## [1] FALSE

An easy mistake is to check equality with =.

4=4 #doesn't work, '=' is for assigning values to variables

We use logicals in if statements.

z <- 3
if (z==3){ 
  print('z is 3') 
}
## [1] "z is 3"

We can also add an action if our condition is FALSE.

if (z==4){ 
  print('z is 4') 
} else {
  print('z is not 4')
}
## [1] "z is not 4"

Factors

Notice that some columns are factors. R turns characters into factors when reading/making tables by default. Factors are a special data type with levels. Factors can be difficult to use, so avoid when possible.

#we can coerce factors to character
d[,1] <- as.character(d[,1])
str(d) #1st column is now character
## 'data.frame':    2 obs. of  3 variables:
##  $ first.names: chr  "Frederick" "Charles"
##  $ last.names : chr  "Banting" "Best"
##  $ num        : int  1 2

We can also prevent R from turning characters into factors by setting a global option

options(stringsAsFactors=FALSE) #applies to the rest of your session
d2 <- data.frame(first.names, last.names, num=1:2)
str(d2) #no factors
## 'data.frame':    2 obs. of  3 variables:

Example data set

Well work with a simulated gene expression matrix, such as from a microarray, with 100 genes & 9 samples, split into 3 groups. We randomly simulate normally distributed values with rnorm(). We make the 1st gene up-regulated in group a.

Were going to treat this data as being on the log2-scale and already processed. For many datasets, there are zeroes or missing values that might need to be imputed; samples need to be normalized to make them comparable and amenable to statistical tests; absent genes need to be removed; sample outliers need to be assessed to examine whether some experimental variables or batch effects need to be accounted for or the samples need to be removed or down-weighted; and trends between a genes mean expression and its variance should be accounted for, especially in RNA-seq data voom.

set.seed(0) #set a seed for random sampling to get reproducible results
gx <- matrix(rnorm(n=900), nrow=100, ncol=9)
#upregulate gene 1
gx[1, 1:3] <- gx[1, 1:3]+3
#sep='' says not to put a space between 'gene' and the numbers
rownames(gx) <- paste('gene', 1:100, sep='')
#3 samples in group 'a' & 3 in group 'b' & 3 in group 'c'
colnames(gx) <- c(paste('a', 1:3, sep=''), paste('b', 1:3, sep=''), paste('c', 1:3, sep=''))

We create matching phenotype & annotation data frames.

pheno <- data.frame(grp=rep(c('a', 'b', 'c'), each=3))
rownames(pheno) <- colnames(gx)

# make random gene symbols: don't worry about how i generate these
annot <- data.frame(symbol=paste0(sample(x=LETTERS, size=100, replace=TRUE), 
                                        sample(x=LETTERS, size=100, replace=TRUE), 
                                        sample(x=LETTERS, size=100, replace=TRUE)),
                    pathway=sample(x=c("oxphos", "glycolysis", "tca cycle", "none"), 
                                   size=nrow(gx), replace=TRUE))
rownames(annot) <- rownames(gx)

Differential expression using base R (not add-on packages)

  1. one gene at a time
  2. for loop
  3. apply function

One gene at a time.

t.test(gx[1, 1:3], gx[1, 4:6])$p.value #1st gene
## [1] 0.02976692
t.test(gx[2, 1:3], gx[2, 4:6])$p.value #2nd gene
## [1] 0.5437477

for loop

This can be automated using a for loop. Well create an empty vector of NAs & assign the t-test p-values to that vector. This loops the variable gene through each of the values between 1 & 100, and applies a t-test on that row.

pv <- rep(NA, times=100)
for (gene in 1:100){
  pv[gene] <- t.test(gx[gene, 1:3], gx[gene, 4:6])$p.value
}

apply

Theres a more advanced & faster way you apply the t.test function to each row.

# MARGIN=1 says to apply to rows
pv <- apply(gx, MARGIN = 1, FUN=function(row.v){ 
  t.test(row.v[1:3], row.v[4:6])$p.value 
})

ezlimma package

You can install ezlimma by following instructions at ezlimma. ezlimmas tutorial is presented as a vignette, like many R packages.

library(ezlimma)
# browseVignettes(package="ezlimma")

Here we create multiple group comparisons, and test them all in one line with limma_contrasts.

contrast.v <- c(AvsB='a-b', AvsC='a-c', BvsC='b-c')
toptab <- limma_contrasts(object=gx, grp=pheno$grp, contrast.v = contrast.v)

toptab is ordered by overall significance of the 3 tests. We round the number of significant digits in toptab to 3 with signif. When we add annotation, we make sure the rows of toptab match the rows of annot using annot[rownames(toptab),]. We write this to diff_gene_stats.csv.

toptab.ann <- data.frame(signif(toptab, 3), annot[rownames(toptab),])
write.csv(toptab.ann, "diff_gene_stats.csv")

In ezlimma, we can also test correlation of each gene to a continuous phenotype in one line of code.

Y <- c(1:9)
toptab.y <- ezcor(object = gx, phenotype = Y, prefix = "Y")

toptab.y is ordered by the significance of the correlation and gives correlation coefficient, p-value, and FDR.

We can plot the top gene, gene27, vs.Y.

plot(gx["gene27",], Y)

For pathway analysis, ezlimma extends limma roast to write linked Excel files, as explained in the ezlimma vignette.

We make a Principal Component Analysis (PCA) plot to see how samples cluster. We use the ezpca function, which is based on the R package ggplot2. So ezpca doesnt print to file, we set name=NA .

Significance histograms

We make a histogram of p-values and false discovery rates (FDR) for each comparison, at signif_hist.pdf.

signif_hist(tab = toptab)

Venn diagram

We can see how many genes with p <= 0.05 the comparisons have in common.

venn.tab <- ezvenn(tab=toptab, name=NA, p.cutoff = 0.05)

This plot doesnt tell us which genes are in common, but venn.tab has this info, which we can view.

Heatmap

We can make a heatmap of the top genes in toptab.ann.

top.genes <- rownames(toptab.ann)[1:10]
ezheat(object = gx[top.genes,], labrows=toptab.ann[top.genes, "symbol"], pheno.df = pheno, name=NA)

By default, the row scaling centers the rows by subtracting the row mean. Then the new mean of every row is zero. This helps see contrast between samples, instead of between rows.

Volcano plot

We make a volcano plot for the 1st comparison, where we label the 3 most significant genes.

ezvolcano(tab=toptab.ann, ntop.sig=3, lab.col="symbol", comparison = "AvsB", name=NA)

We can also make a volcano plot for all comparisons, volcanoes.pdf.
